0.            Introduction

back to contents

ManuWare is a high-performance software tool used to set-up support of Kistler measuring chains for a wide range of different measuring tasks in the manufacturing sector. The following Kistler devices are currently supported:

Amplifiers

·         ICAM Type 5073A... (ICAM: Industrial Charge Amplifier for Manufacturing)

Transmitters

·         Strain transmitter Type 9238A...

·         Press force transmitter Type 9337A...

Sensors

·         All Kistler piezoelectric sensors in conjunction with the charge amplifiers listed above

ManuWare Version 1.00 can perform the following functions:

·         Administration of ManuWare projects (create, save, load etc.)

·         Administration of supported devices in Project Explorer

·         Configuration of devices and measuring channels

·         Scaling of measuring channels to known measurands

·         Real-time display of current device status

·         Control of supported devices by the PC via the serial interface (measure reset, range switching)

·         Display of measurement data with reduced data rate for activation of a measuring chain

·         Definition of a display interface specially adjusted for the measuring task

·         Definition and administration of user profiles

1.Program Interface – Description

back to contents

After the initial program start, ManuWare will appear as follows:

Fig. 1:           ManuWare main window after first program start. The various program areas are surrounded by blue dotted lines.

The various program areas (blue dotted lines) have the following tasks:

  • Title, menu, tool and status bars are known from Windows and have the same functions. The main part of the window includes the following three areas: navigation frame, graphic display frame and information frame.
  • Navigation frame:
    Includes 'Project Explorer', which lists the devices in a project. Project Explorer permits the following functions: Configuration of devices and channels, activation for measurement; software range switching, overview of device status.
  • Graphics display frame:
    Any number of worksheets can be defined here. This is where graphic display objects are placed (oscilloscope, XY plot, digital multimeter etc.).
  • Information frame:
    The information frame contains two index cards with device and system information. The current status of the individual devices or channels is shown on the 'Status' tab. Warnings, error messages and/or the entire protocol between ManuWare and the connected devices, if required, are listed on the 'Messages' tab.
  • Display objects bar:
    All available display objects are listed here. The display objects can be moved from here to a worksheet using the drag-and-drop function.

1.2.        Worksheets – Structured Display of Measurements

back to contents

Measurement results are displayed via the display objects, which are arranged on the worksheets. When a new project is created, a single empty worksheet will appear in the graphic display frame (see Fig. 1). Several different display objects can be shown on a worksheet. Additional worksheets can also be added.

     Add worksheet                                   A new worksheet will open in the graphic display frame if you click on the 'Add Worksheet' button.

Any number of worksheets can be set up here. New worksheets are called 'Worksheet n' as standard, where n stands for a consecutive number.

Fig. 7:           Click the right-hand mouse button on the tab index of the worksheet to open the context menu. New worksheets can be added, the current worksheet can be deleted or its name can be changed. The same menu can also be found in the menu bar under 'worksheets'.

The worksheet name that is given automatically can be changed in the context menu using 'Rename worksheet'.

1.3.        Display Objects

back to contents

Display objects show measurement results and curves. The following three display objects are currently available:

     Cycle display object                           This display object is only updated after measurement is complete (Reset). It is particularly suitable for displaying two-channel measurements (force-displacement, torque-angle, etc.). It is therefore most useful for displaying CoMo Net® results. A measuring cursor and a zoom (drag the mouse while holding down the Ctrl button) are also available. Several channels can be displayed at the same time in the cycle display object.

     Oscilloscope display object                This display object shows the measuring signal depending on time. It is updated in real-time and is used for initial checking of measuring channels.

     Digital multimeter display object       Using this display object, the signals measured are displayed in real-time numerically and as a bar diagram.

2.            First Simple Measurement – Step by Step

back to contents

2.1.        Preparation

back to contents

To make a simple measurement (for example activation of a measuring chain), three conditions must have been fulfilled:

1.      ManuWare must be installed on the PC.
Follow the installation instructions on the installation CD.

2.      The measuring chain to be checked (or several measuring chains) must be mounted on your machine and switched on.
The instructions can be found in the instruction manual for your sensors, amplifiers or transmitters. Please ensure that the mechanical installation of the sensors is correct, that the sensors are correctly connected to the charge amplifier using high insulation connection cables and that the supply, signal and control cables are correctly connected to the charge amplifier.

3.      The measuring chain amplifiers or transmitters must be connected to the PC.
For the amplifier and transmitter models currently supported, this is done via the RS-232C interface. You can use the PC's COM ports directly, or the COM ports on a USB serial hub.

2.2.             Create New Project – 'New project' (Ctrl + N)

back to contents

The first time ManuWare is started up, a new, empty project with the name 'Project 1' will be created automatically. If there is already an existing project, a new empty project can be opened at any time using the 'Open project...' function. You will, of course, be asked whether the existing project should be saved, if it has not already been saved.

2.3.         Find/Connect Device – 'AutoScan'

back to contents

In order to be able to communicate with the devices connected, ManuWare needs to know which type of device is connected to which serial interface. This can be done manually using the 'Add device…' function. However, the device type and the corresponding interface must be known.

The 'AutoScan' function is the easiest alternative. After ManuWare has been started up for the first time or after a new project has been created, all you need to do is click on the 'AutoScan' button and ManuWare will independently search all available COM ports for defined devices (currently Type 5073A..., 9238A... and 9337A...).

Devices found are immediately listed in the 'Project Explorer', while the automatic device scan will continue to run in the background (search time approx. 10s.). When the device scan ends, the message Scanning Completed will appear.

Fig. 12:          Once the automatic scan is complete, the devices found are displayed in the Project Explorer and the end of the scan will be reported in a dialog box.

The devices identified (and their measuring channels) are now available for measuring.

back to contents

Three different display objects are available in the current version. They can be found in the display objects bar (see 1.3 Display Objects). The display objects required may be placed on a worksheet with the mouse, using the drag-and-drop function.

2.8.        Example of scaling

back to contents

A scaling factor (m or x) is determined using the scaling procedure. The scaling factor is used for interpretation of the voltage output signal of a transmitter or amplifier. The voltage signal in volts is multiplied by the scaling factor, in order to get the measured input size. The scaling factor is saved in the transmitter or amplifier and can be read by the data processing system (data acquisition, display system) and used for interpreting the voltage signal. The voltage output itself is not affected by the scaling.

However, if range settings are changed, the relationship between measurand and voltage signal and therefore also the scaling factor will change. The scaling factor is automatically adjusted if the range settings are changed.

Sensors and transmitters are often used for indirect measurements. That means that the relevant measurand (in Fig. 19: Force) is derived from the measured measurand (in Fig. 19: strain).

Fig. 19:         Indirect measurement – hand press example. In order to measure force indirectly via strain scaling also has to be made.

To illustrate the scaling function more clearly, the transfer model of a measuring chain is shown in Fig. 20. Using the example of the hand press from Fig. 19, Min is the force on the press plate and Xin is the strain on the C-structure.

The scaling factors x or m are given in the 'Channel Properties' dialog box under the heading Scaling factor. For an 'unscaled' measuring chain, factor x is given, which can be used to calculate the internal measurand Xin from the output voltage signal Uout. The scaling factor x of a strain transmitter is shown in the example in Fig. 21.

Fig. 21:          Display of scaling factor in 'Channel Properties' dialog box (in this instance scaling factor x for an unscaled strain transmitter).

If the output voltage signal Uout is multiplied by the factor m, you will obtain the input measurand value as long as the measuring chain has been correctly scaled. The Scaleing function has to be used to determine the factor m. For the scaling process, the input measurand Min firstly has to be measured with a reference measuring chain. In the hand press example, the force on the press plate could, for example, be measured using a press force sensor or transmitter (see Fig. 22).

Fig. 22:          Example of a strain transmitter on a hand press. By default (factory setting), the strain transmitter is only scaled with factor x. That means that in the scaled view (center) the internal measurand is set. Left: Voltage output of the strain transmitter; center: Display of the strain using the scaling factor for the internal measurand x; right: Force signal of a reference measuring chain (here press force transmitter Type 9337A)

The Scaling  is carried out as follows:

Fig. 23:          'Scaling' dialog box with the five steps required.

  1. Pre-selecting a new unit: The unit that is valid for the measuring channel after scaling can be selected here.
  2. Starting the scaling measurement (using the toggle button Start measurement in the symbol bar): After the start of the measurement, the output voltage Uout in the current output voltage field is shown. The system is then mechanically loaded with the nominal force. The user is now free to choose the output voltage that is wanted to scale the system. The current output voltage should preferably amount to approx. 80 % of the full scale. If the output voltage is below 20 % of the full scale, a warning will appear: Voltage signal too low for reliable scaling. In this case, the scaling process can be interrupted and the amplification can be improved via the range setting (see 2.7).
  3. Acquire value: When the mechanical system is stably loaded with the nominal force, the current output voltage can be acquired.
  4. Corresponding input measurand (specifically: corresponding force): The corresponding input measurand is read from the reference measuring chain and given in the 'Scaling' dialog box of the measured output voltage.
  5. Setting the measurement range: Finally, the maximum force range is defined and the full scale output voltage is selected. This process does not actually belong directly to scaling (determining scaling factor). However, because scaling can result in irregular range settings, it is sensible to do the range settings again after every scaling.

The new settings become active after confirming with OK. In some cases, the new parameters still need to be sent to the device connected using Send.

Fig. 24:          'Channel Properties' dialog box after scaling. The parameters surrounded in red may have been changed during scaling.

After successful scaling, a force can be measured directly using the strain transmitter in the present example. The more accurate and careful the scaling process was carried out, the better the consistency. Because the measurand was measured in two different places and possibly in different ways, no two curves will be the same, especially with dynamic applications.

Fig. 25:          After scaling, the scaled sensor (here Type 9238A) and the reference measuring chain (here Type 9337A) show the same measurement result.

2.9.        Save Project – 'Save Project' (Ctrl + S)

back to contents

The project should be saved now at the latest because the measurement has been definitively set up. This is done as usual using the menu 'File >> Save Project' or 'File >> Save Project as...'. The current state of the project will be saved in a file with the extension *.mwproj. It includes all information on the connected devices, their settings and arrangements and settings of the display objects.

3.            Backup/Restore Functionality

back to contents

Using the functions described above, ManuWare allows the settings of a device or an entire application to be saved and, if required, replayed. Proceed as follows:

1.      A successfully running project is saved after activation (see above) under an appropriate name (for example backup_projekt1_080512.mwproj).

2.      Restore is required if the device parameters have been accidentally changed or if a device has failed (replacement). From the point of view of the application (that is ManuWare), both cases are identical.

3.      You must make sure that the relevant device is switched on.

4.      When the saved project is loaded in ManuWare, the identified devices will be connected. The connection can also be made manually, if the device has been connected to another interface in the meantime or if the device is defined as not connected in the project.

5.      When the 'Properties' dialog box is activated, ManuWare recognizes that the parameters in the device and in the project are different. This is shown by a message under the synchronization bar: parameter sets are different!

Fig. 26:          If the parameters in the project and in the device are different, this is shown by a message in the 'Channel Properties' dialog box.

6.      There is now the option of sending the data from ManuWare to the device or loading the data from the device into ManuWare. The first option has been chosen in the present case.

7.      Once the parameters have been sent to the unit, the device is now in exactly the same status as it was when the backup was made.
